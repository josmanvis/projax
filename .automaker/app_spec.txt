  <project_name>PROJAX</project_name>

  <overview>
    PROJAX is a comprehensive cross-platform software project management suite designed for developers to track, manage, and interact with their local development projects. It provides a unified interface across multiple platforms including CLI, Terminal UI (TUI), Desktop Application, Web UI, REST API, and IDE extensions (VS Code, Cursor, Windsurf). The core purpose is to help developers track multiple local projects from different directories, automatically detect test files and frameworks, manage port conflicts, run scripts with intelligent selection, organize projects with tags and descriptions, navigate quickly between projects, and monitor running processes. PROJAX supports multiple project types including Node.js, Python, Rust, Go, and Makefile-based projects, making it a versatile tool for polyglot development environments.
  </overview>

  <development_guidelines>
    <guideline>Use pnpm as the package manager with workspace protocol for internal dependencies</guideline>
    <guideline>Monorepo structure managed with Turbo for parallel builds</guideline>
    <guideline>TypeScript for all packages with strict typing</guideline>
    <guideline>ESLint for code linting with TypeScript plugin</guideline>
    <guideline>Jest for unit testing across packages</guideline>
    <guideline>Husky and lint-staged for pre-commit hooks</guideline>
    <guideline>Semantic versioning with automated version bumping scripts</guideline>
    <guideline>Package-specific builds: core and api must build before cli</guideline>
    <guideline>Use workspace:* protocol for internal package dependencies</guideline>
    <guideline>Follow existing component patterns for Desktop app (React components with CSS modules)</guideline>
    <guideline>API endpoints should return appropriate HTTP status codes with JSON error responses</guideline>
    <guideline>CLI commands should provide helpful output and support both interactive and non-interactive modes</guideline>
    <guideline>Support cross-platform operations (macOS, Linux, Windows)</guideline>
    <guideline>Database operations go through the API server, core package provides client abstraction</guideline>
  </development_guidelines>
